# **Emoji Sarcasm Detection in Messages**

Sarcasm and irony detection is a well-known NLP challenge, made harder on social media where meaning is often conveyed through emojis rather than words alone. Most existing approaches strip emojis during preprocessing, ignoring a signal that frequently acts as a sentiment amplifier or irony marker.
This project investigates whether explicit emoji handling via Emoji Sentiment Ranking (ESR) lexicon features improves binary sarcasm/irony classification over text-only baselines. We evaluate across TweetEval (irony) and iSarcasmEval (sarcasm), comparing four systems: TF-IDF + Logistic Regression, pre-trained RoBERTa-irony, pre-trained BERT, and a fine-tuned Twitter-RoBERTa each tested under three emoji-handling modes (removed, retained, replaced with descriptors).
Research question: Does incorporating explicit emoji semantics improve sarcasm/irony detection, and at which stage of the pipeline does it matter most?.

All datasets are downloaded automatically by running the notebook. No manual downloads required.

| Dataset | Source | Purpose |
|---|---|---|
| TweetEval Irony | HuggingFace `cardiffnlp/tweet_eval` | Training + in-domain evaluation |

## Notebook Structure

The notebook is divided into the following stages:

| Stage | Description |
|---|---|
| **Stage 1 Setup** | Downloads datasets, builds ESR dictionary, standardises schema, extracts emoji features, applies preprocessing modes (T/K/D) |
| **Stage 2 Baseline + Pre-trained** | TF-IDF + Logistic Regression baseline (text-only and text+ESR); inference-only evaluation of `bertweet-irony` and `twitter-roberta-base-irony` |
| **Stage 3 Fine-tuning** | Fine-tunes `cardiffnlp/twitter-roberta-base` on TweetEval under three emoji modes (T, K, D); evaluates on TweetEval test set |
| **Stage 4 Evaluation** | Quantitative evaluation (Macro-F1, Accuracy, Precision, Recall, Confusion Matrix) on TweetEval and iSarcasmEval; final comparison across all models |
| **Stage 5 Error Analysis & CLI** | Qualitative inspection of 10 misclassified examples from best model; categorisation into error types |

## Emoji Handling Modes

| Mode | Description | Example |
|---|---|---|
| **T** (Strip) | Remove all emojis from text | `"Great job 😒"` → `"Great job"` |
| **K** (Keep) | Keep raw Unicode emojis | `"Great job 😒"` → `"Great job 😒"` |
| **D** (Demojize) | Convert emoji to text alias | `"Great job 😒"` → `"Great job :unamused_face:"` |

## Results Summary

| Model | Stage | TweetEval F1 | iSarcasm F1 |
|---|---|---|---|
| TF-IDF text-only | Baseline | 0.611 | 0.474 |
| TF-IDF + ESR | Baseline | 0.619 | 0.514 |
| BERTweet-irony | Pre-trained | 0.819 | 0.613 |
| RoBERTa-base-irony | Pre-trained | 0.711 | 0.656 |
| RoBERTa fine-tuned (T) | Fine-tuned | 0.739 | 0.719 |
| RoBERTa fine-tuned (D) | Fine-tuned | 0.733 | 0.709 |
| **RoBERTa fine-tuned (K)** | **Fine-tuned** | **0.751** | **0.718** |

> iSarcasm F1 scores for fine-tuned models use a raised classification threshold to account for the dataset's significant class imbalance (86% non-sarcastic): **0.85** for Mode T, **0.90** for Modes K and D.

---

## Key Findings

- Mode K (keep raw emojis) achieves the highest TweetEval F1 (0.751), outperforming Mode T (0.739) on the in-domain test set and confirming that raw emoji signals aid sarcasm detection
- On iSarcasmEval, Mode T marginally outperforms Mode K (0.719 vs 0.718), suggesting emoji benefit is less consistent out-of-domain
- All three fine-tuned models outperform both pre-trained baselines on iSarcasmEval, demonstrating strong generalisation to unseen sarcasm data
- Mode D (demojize) ranks lowest on both datasets, indicating that converting emojis to text aliases does not add meaningful signal over keeping them raw
- The LR baseline confirms that deep contextual representations are essential — TF-IDF features alone are insufficient for sarcasm detection in social media text
